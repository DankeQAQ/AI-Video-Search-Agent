# 视频搜索 Agent — 产品需求说明（PRD）

## 1. 产品概述

基于自然语言的**影视/动漫资源检索 Agent**：用户用口语描述想看的内容（作品名、季、集、篇章等），系统解析意图并对接搜索/元数据能力，返回可播放或可追溯的结果。

## 2. 目标用户

- 记不住精确片名拼写、但能用中文描述的用户  
- 需要按「季/集/篇章」定位单集的用户  

## 3. 核心能力

| 模块 | 说明 |
|------|------|
| 意图解析（大脑） | 将模糊输入结构化为作品名、标准季数、集数、篇章名（含官方英文篇章名，如 Book of Circus） |
| 检索与排序 | 结合结构化意图查询索引或外部源，排序与去重 |
| API 服务 | 提供 HTTP 接口，便于客户端或自动化调用 |

## 4. 非目标（当前阶段）

- 不提供盗版资源链接或绕过版权的指引  
- 不保证覆盖全部片库（依赖后续数据源接入）  

## 5. 技术栈

- **语言**：Python 3.9+  
- **Web**：FastAPI、Uvicorn  
- **Agent / LLM**：LangChain（编排）、OpenAI API（及其他兼容端点）  
- **数据校验**：Pydantic  
- **HTTP 客户端**：httpx（调用外部搜索/元数据 API）  

## 6. 里程碑（建议）

1. 环境初始化与 API 骨架（本阶段）  
2. 意图解析器稳定化与评测集  
3. 搜索适配层与结果融合  

## 7. 接口约定（草案）

- `GET /health`：存活探针  
- `POST /api/v1/parse`：自然语言 → 结构化意图（随实现迭代）
